Cocos2D(一)-Win7和eclipse中配置开发环境

By Never

发表于 Apr 7 2014

文章目录

##cocos2d介绍##
cocos2d是一个基于MIT协议的开源框架，用于构建游戏、应用程序和其他图形界面交互应用。可以让你在创建自己的多平台游戏时节省很多的时间。
从本质上来说，Cocos2d是一个图形引擎，封装了复杂的图形接口，通过抽象出精灵。动作等概念，降低了游戏开发难度，简化了开发过程。Cocos2d-x为保证游戏能方便的移植到不同平台上，又在此基础上做了很多扩展，包括一套Objective-C风格的基础类系、平台无关的多点触碰协议、重力感应和音频系统等。

Cocos2D也拥有几个主要版本。包括Cocos2D-iPhone、Cocos2D-X，以及被社区普遍看好的Cocos2D-HTML5和JavaScript bindings for Cocos2D-X。同时也拥有了非常优秀的编辑器（独立编辑器），例如SpriteSheet Editors、Particle Editors 、Font Editors 、 Tilemap Editors。

##cocos2d特点##

  1. 易于使用:游戏开发者可以把关注焦点放在游戏设置本身，而不必消耗大量时间学习晦涩难懂的OpenGL ES，此外，Cocos2D还提供了大量的规范。
  2. 高效：Cocos2D基于OpenGL ES进行图形渲染，从而让移动设备的GPU性能发挥到极致。
  3. 灵活：方便扩展，易于集成第三方库。
  4. 免费：基于MIT协议的免费开源框架，用户可以放心使用，不用担心商业授权的问题。
  5. 社区支持：关心Cocos2D的开发者自发建立了多个社区组织，可以方便的查阅各类技术资料。

##主要功能##

  1. 流程控制（Flow control）：非常容易地管理不同场景（scenes）之间的流程控制
  2. 精灵（Sprites）：快速而方便的精灵
  3. 动作（Actions）：告诉精灵们该做什么。可组合的动作如移动（move）、旋转（rotate）和缩放（scale）等更多
  4. 特效（Effects）：特效包括波浪（waves）、旋转（twirl）和透镜（lens）等更多
  5. 平面地图（Tiled Maps）：支持包括矩形和六边形平面地图
  6. 转换（Transitions）：从一个场景移动到另外一个不同风格的场景
  7. 菜单（Menus）：创建内部菜单
  8. 文本渲染（Text Rendering）：支持标签和HTML标签动作
  9. 文档（Documents）：编程指南 + API参考 + 视频教学 + 很多教你如何使用的简单测试例子
  10. BSD许可：尽管用就是了
  11. 基于Pyglet：没有外部的依赖
  12. 基于OpenGL：支持硬件加速

##Win7 eclipse下配置开发环境##
最近开始学习cocos2d，选择cocos2d-x版本，想要在eclipse中配置开发环境。下面就开始我们的配置之旅。

  1. 首先我之前已经配置好eclipse的android的开发环境。
  2. 安装NDK。
  3. 下载安装python，然后配置环境变量，添加python的路径到path。
  4. 解压缩cocos2d-x-2.2.3。
  5. 复制python.bat（另外下载）到cocos2dx解压缩的根目录下\tools\project-creator下，然后双击运行，输入项目名称级包标识名，创建项目。然后可以到\project目录下看到刚刚创建的项目。
  6. 将刚刚创建的cocos2dx项目导入到eclipse中，选择创建的android版本（后缀名proj.android）。
  7. 将刚刚创建的项目Resource目录下的资源文件全部复制到导入的android项目assets文件夹下，同时将\cocos2dx\platform\android\java\src目录下的org文件复制到当前项目的src文件夹中。然后refresh
  8. 右键选择项目属性（Properties），选择C/C++ builder，填入NDK的ndk.build.cmd路径；然后选择Environment填入一些环境：添加COCOS2DX（cocos2dx解压缩的目录）、NDK_ROOT（NDK解压缩的根目录）、NDK_MODULE_PATH（解压缩根目录，类似于D:/cocos2d-x-2.2.3/cocos2dx/platform/third_party/android/prebuilt）
  9. 设置资源的环境变量。选择Resource，添加COCOS2DX，也就是cocos2dx的解压缩的根目录。
  10. 右键bulid project
  11. 运行项目

##文件目录##

  1. assets：资源文件。
  2. AppDelegate:程序入口类。
    1. bool applicationDidFinishLaunching（）：应用程序启动之后调用这个方法。
    2. void applicationDidEnterBackground():程序将要进入后台时调用。
    3. void applicationWillEnterForeground():程序回到前台时调用。
  3. HelloWorldScene：程序主要绘制类。
    1. static CCScene* scene():静态方法创建场景，因为改场景中只有一个层。
    2. bool init()：初始化话HelloWorld类

##配置过程中出现的一些问题##

  1. Android NDK: WARNING: Ignoring unknown import directory: C:\cocos2d-2.0-x-2.0.3 Android NDK: jni/Android.mk: Cannot find module with tag ‘cocos2dx’ in import path jni/Android.mk:19: * Android NDK: Aborting. . Stop.

    在andorid.mk文件中添加

    $(call import-add-path, D:/cocos2d-x-2.2.3)
$(call import-add-path, D:/cocos2d-x-2.2.3/cocos2dx/platform/third_party/android/prebuilt)

    注意路径是使用的”/“而不是”\”

  2. Error: Program “bash” is not found in PATH

    如果是用cygwin+NDK+eclipse环境生成的工程，工程已经添加了native super。

    1、把cygwin/bin加入系统环境变量Path下

2、右击工程-->properties-->C/C++ Build  确认右边视图选项Build command：ndk-build

3、右击工程-->properties-->C/C++ Build -->ToolChain Edit
  确认右边视图的选项：Current toolchain：No Toolchain
                  Current Builder：Android Builder    (如果还是不行，可以换成Gnu Make Builder试试)
